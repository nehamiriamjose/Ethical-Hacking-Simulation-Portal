What is XSS?

Cross-Site Scripting (XSS) is a vulnerability where an attacker injects malicious scripts into trusted websites. When a user visits the compromised page, the browser executes the script, thinking it's part of the legitimate site.

In this simulation, we are demonstrating **Stored XSS**—the most dangerous type, where the script is permanently saved on the server.

Simulation Vulnerability

VibeSocial's comment system is vulnerable because it uses the | safe filter in its template. This tells the system to render HTML tags instead of treating them as plain text.

<p>{{ comment.text | safe }}</p>

Because of this, an attacker can post a <script> tag, and it will run for every single user who views the feed.

Exploitation Guide

To confirm the hack, you must execute a JavaScript alert. The system monitors for this alert and will decrypt the flag upon detection.

<script>alert('Hacked')</script>

Paste this into the comment box and click Send to reveal the flag.
